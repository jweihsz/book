####1.2.2 对话
如果有一句代码:
```
var a = 2;

```
编译器的处理过程如下：
编译器查询在该作用区域是否已经有了var a的声明，如果已经声明了，就跳过。如果没有声明，那么会重新声明一个变量a。接下来会为引擎生成运行时所需要的代码，这些代码被用来处理a=2时的赋值。引擎在运行时，首先查找在该作用区域中是或否有变量a的声明，如果有，则进行赋值操作，如果没有找到，则抛出错误。


####1.3 作用区域的嵌套
这里只要记住一点就好，就是在查找变量的时候，它是从内部向外部进行查找，如果到达了最外层，则不再进行查找。


####1.4异常 
在执行LHS和RHS的查询过程中，这两种的结果是不同的。执行RHS查询时，如果遍历所有的作用区域，都没有找到，那么就直接抛出ReferenceError错误提示，如果RHS在所有区域中没有找到，那么在非严格模式下，会主动进行创建该变量，并把其作为全局变量。如果对变量进行不合理的操作，那么是会抛出TypeError错误。
